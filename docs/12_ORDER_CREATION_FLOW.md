# Order Creation Flow

## Формат
Создание ордера реализовано как multi-step мастер (wizard).

## Цель
Дать пользователю возможность собрать заказ на трафик самостоятельно:
- выбрать источники
- задать бюджет
- задать ожидаемые параметры
- отправить в оплату / ожидание подтверждения менеджером

## Шаги (примерный базовый поток)
### Step 1: Основа
- Название ордера (опционально)
- Цель (leads/installs/sales/traffic)
- Ниша (опционально)
- Гео (обязательно)

### Step 2: Источники
- Выбор источников (TikTok/FB/YouTube/Telegram Ads и т.д.)
- Для каждого источника: чекбокс включения

### Step 3: Бюджет
- Общий бюджет (обязательно)
- Дневной лимит (опционально)
- Срок (дней) или дата окончания (опционально)

### Step 4: KPI / Опции
- Модель: CPA / CPL / CPM / CPC (опционально)
- Значение KPI (опционально)
- Комментарии и пожелания (опционально)

### Step 5: Распределение бюджета
Поддерживаем 2 режима:
1) Авто-распределение (по умолчанию)
2) Ручное распределение (проценты / лимиты по источникам)

Валидации:
- сумма процентов = 100%
- лимиты не превышают общий бюджет

### Step 6: Подтверждение
- Сводка ордера
- Кнопка: «Оплатить» или «Сохранить и оплатить позже»

## Валидации (минимум)
- Гео не пустое
- Общий бюджет > 0
- Выбран хотя бы 1 источник (если не включён авто-режим выбора источников)

## После сборки
- Ордер сохраняется как draft/pending_payment
- Ордер можно оплатить в течение 48 часов
