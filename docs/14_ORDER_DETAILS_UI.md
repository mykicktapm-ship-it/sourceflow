# Order Details UI

## Экран «Детали ордера»
Цель: дать пользователю контроль и прозрачность по заказу.

## Блоки
### Header
- Название / цель
- Статус (badge)
- Общий бюджет

### Прогресс
- Потрачено (spent)
- Остаток
- Прогресс-бар

### Источники
Список источников с показателями:
- статус источника (active/review/paused)
- доля бюджета
- потрачено

### KPI
- выбранный KPI (если задан)
- фактические значения (если доступно)

### Логи / события (упрощённо)
- created
- payment pending/paid
- manager confirmed
- started/paused/completed

## Действия
Зависят от статуса:
- draft/pending_payment: Оплатить, Редактировать, Удалить
- paid: Ожидать подтверждения менеджера
- active: Пауза, Изменить бюджет (если разрешено), Связаться с менеджером
- paused: Продолжить, Отменить
- completed: Смотреть отчёт, Создать похожий
- expired: Создать заново
